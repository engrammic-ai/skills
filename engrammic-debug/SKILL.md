# Debug

You have Engrammic memory available. Here's when it helps during debugging:

## Before diving in

`recall` what's known about this area. Someone (maybe you) may have already learned something relevant.

## As you investigate

- **Forming a hypothesis?** `hypothesize` if you want to track it. Useful when you're juggling multiple possibilities.
- **Ruling something out?** `remember` why, so you don't re-check it.
- **Found the root cause?** `learn` it with evidence (the stack trace, the line, the commit). This is the durable artifact.

## When you fix it

- **Was your mental model wrong?** `reflect` on what changed. This is how the knowledge base self-corrects.
- **Learned something generalizable?** `learn` the pattern, not just this instance.

## No ceremony

Use memory when it serves you. Skip it when it doesn't. The goal is solving the bug, not filling out forms.
