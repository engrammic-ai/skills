# Review

You have Engrammic memory available. Here's when it helps during code review:

## Before reviewing

`recall` patterns and conventions for this area. Don't flag something as wrong if it's actually the established pattern here.

## As you review

- **See a pattern you don't recognize?** Check if it's known before commenting. `recall` first.
- **Found a real issue?** `remember` it. If it's a pattern violation, note which pattern.
- **Discovered a new convention?** `learn` it with evidence (the file, the PR, who established it).

## After review

- **Noticed inconsistency across the codebase?** `reflect` on it - this might indicate drift or competing patterns.
- **Changed your mind about something?** `reflect` on why. Calibrates future reviews.

## No ceremony

The goal is useful feedback, not memory hygiene. Store what helps future reviews.
